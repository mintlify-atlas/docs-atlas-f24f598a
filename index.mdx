Bencher is a header-only C++23 library for writing precise, reproducible benchmarks. It collects hardware performance counters, applies rigorous statistical analysis, and produces actionable output — all without any linking step.

## Why Bencher?

<CardGroup cols={2}>
  <Card title="Header-only" icon="file-code">
    Single `#include` — no build system integration beyond adding the include path.
  </Card>
  <Card title="Hardware counters" icon="microchip">
    Reads CPU cycles, instructions, branches, and branch misses directly from hardware on macOS, Linux, and Windows.
  </Card>
  <Card title="Rigorous statistics" icon="chart-line">
    Runs until a 95% confidence interval converges, then reports median and MAD — not noisy means.
  </Card>
  <Card title="Multiple output formats" icon="file-export">
    Console output, Markdown reports, SVG bar charts, and optional JSON for CI pipelines.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Add Bencher to your CMake project">
    ```cmake CMakeLists.txt
    include(FetchContent)
    FetchContent_Declare(
        bencher
        GIT_REPOSITORY https://github.com/stephenberry/bencher.git
        GIT_TAG main
    )
    FetchContent_MakeAvailable(bencher)

    target_link_libraries(your_target PRIVATE bencher::bencher)
    ```
  </Step>
  <Step title="Write your first benchmark">
    ```cpp benchmark.cpp
    #include "bencher/bencher.hpp"
    #include "bencher/diagnostics.hpp"

    int main()
    {
       bencher::stage stage{"My Benchmarks"};

       stage.run("Algorithm A", [&] {
          // code to benchmark
          return bytes_processed;
       });

       bencher::print_results(stage);
       return 0;
    }
    ```
  </Step>
  <Step title="Build and run">
    ```bash
    cmake -B build -DCMAKE_BUILD_TYPE=Release
    cmake --build build
    ./build/benchmark
    ```
  </Step>
  <Step title="Interpret the results">
    Bencher prints throughput, hardware counters, and a comparison bar chart automatically. See [Performance Metrics](/concepts/metrics) for a full explanation of every field.
  </Step>
</Steps>
