description: "Add Bencher to your C++ project."

<Tabs>
  <Tab title="CMake FetchContent">
    The recommended way to add Bencher is via CMake's built-in `FetchContent` module. Add the following to your `CMakeLists.txt`:

    ```cmake CMakeLists.txt
    include(FetchContent)
    FetchContent_Declare(
        bencher
        GIT_REPOSITORY https://github.com/stephenberry/bencher.git
        GIT_TAG main
    )
    FetchContent_MakeAvailable(bencher)

    target_link_libraries(your_target PRIVATE bencher::bencher)
    ```

    **Choosing a `GIT_TAG`:**

    - `main` — always fetches the latest commit. Good for development, but the build is not reproducible across machines.
    - A specific tag (e.g., `v0.2.1`) — pins the exact version. Recommended for production projects and CI environments where reproducibility matters.

    **Enabling JSON output:**

    JSON support depends on [Glaze](https://github.com/stephenberry/glaze). To enable it, set the option before calling `FetchContent_MakeAvailable`:

    ```cmake CMakeLists.txt
    include(FetchContent)
    FetchContent_Declare(
        bencher
        GIT_REPOSITORY https://github.com/stephenberry/bencher.git
        GIT_TAG main
    )
    set(BENCHER_ENABLE_JSON ON)
    FetchContent_MakeAvailable(bencher)

    target_link_libraries(your_target PRIVATE bencher::bencher)
    ```

    When `BENCHER_ENABLE_JSON` is `ON`, CMake automatically fetches the Glaze library and links it into `bencher::bencher`. No additional steps are required.
  </Tab>

  <Tab title="Manual integration">
    If you are not using CMake, copy the `include/bencher` directory into your project and add it to your compiler's include path.

    ```bash
    cp -r /path/to/bencher/include/bencher your_project/include/bencher
    ```

    Then tell your build system where to find the headers. With CMake:

    ```cmake CMakeLists.txt
    target_include_directories(your_target PRIVATE path/to/include)
    ```

    With a compiler directly:

    ```bash
    g++ -std=c++23 -I path/to/include benchmark.cpp -o benchmark
    ```

    Because Bencher is header-only, there is nothing to compile or link beyond the include path.

    <Note>
      Manual integration does not receive automatic updates. You will need to re-copy the headers whenever you want to upgrade to a newer version.
    </Note>
  </Tab>
</Tabs>

## Requirements

| Requirement | Minimum version | Notes |
|-------------|-----------------|-------|
| C++ standard | C++23 | Required by the library |
| GCC | 13 | Tested on CI with GCC 13 and 14 |
| Clang | 17 | Tested on CI with Clang 17, 18, and 19 |
| MSVC | Latest | Tested on Windows CI |
| Apple Clang | Latest | Tested on macOS CI including Apple Silicon |
| CMake | 3.24 | Required for the FetchContent method only |

<Note>
  Manual integration only requires an include path — no CMake or build system integration is needed. The C++23 requirement still applies regardless of how you integrate the library.
</Note>
